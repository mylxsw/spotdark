# Spotdark 功能梳理与开发计划

> 文档日期：2026-04-21  
> 应用版本：初始原型（基于 superflux 重命名为 Spotdark）

---

## 一、已实现功能

### 1. 全局快捷键触发
- 默认快捷键：Command+Space（若被系统占用则自动降级为 Option+Space）
- 快捷键注册失败时弹出警告对话框提示用户
- Menu Bar 入口作为备选触发方式

### 2. 应用发现与索引
支持三种数据源，按需组合使用：

| 数据源 | 实现文件 | 特点 |
|--------|----------|------|
| 目录扫描 | `DefaultAppProvider` | 递归扫描 `/Applications`、`~/Applications`、`/usr/local/Caskroom` 等 |
| Spotlight 实时流 | `SpotlightIndexStream` | 异步流，支持初始快照 + 增量更新 |
| Spotlight 元数据查询 | `MetadataAppProvider` | 同步查询，适合冷启动 |

- 去重：以 Bundle URL 为键的集合去重
- 排序：按显示名称字母序

### 3. 搜索与排名算法
- 前缀匹配（得分 0，最高优先）
- 词边界匹配（得分 1）
- 子串匹配（得分 2）
- 使用频率加权：启动次数越多，得分越低（排名越靠前）
- 同分按字母序排列
- 默认最多返回 20 条结果，80ms 防抖

### 4. 内建命令系统
- `open-settings`：打开 macOS 系统设置
- `quit`：退出应用
- 可扩展的 `CommandRegistry` 支持运行时注册新命令

### 5. UI 界面
- 无边框浮动面板（NSPanel），720×420，屏幕居中
- 毛玻璃背景（`.ultraThinMaterial`）+ 圆角（16pt）+ 1pt 描边
- 浮于所有 Space 之上，失焦自动隐藏
- 搜索栏：放大镜图标 + 实时输入 + 清空占位符
- 结果列表：28×28 图标 + 标题 + 类型徽章 + 高亮匹配词
- 平滑动画选中高亮
- 键盘导航：↑↓ 移动、Enter 启动、Escape 关闭

### 6. 使用频率持久化
- 记录每个应用的启动次数（Bundle URL 为键）
- 存储于 `UserDefaults`（key: `usage.app.open.count`）
- 线程安全：`NSLock` 保护

### 7. 图标与名称缓存
- 应用图标：`NSCache`，上限 512 条，缩放至 28×28
- 显示名称：`NSCache`，上限 2048 条

### 8. Menu Bar 集成
- 应用以 `.accessory` 策略运行（不显示 Dock 图标）
- Menu Bar 菜单：Show Launcher（L）/ Quit（Q）

### 9. 测试覆盖
- SearchEngine 排名与得分
- AppProvider 目录扫描
- CommandRegistry 注册与检索
- LauncherViewModel 状态管理
- Spotlight 流增量发射

## 二、已知局限与技术债

| 问题 | 位置 | 说明 |
|------|------|------|
| MetadataAppProvider 使用阻塞 Semaphore | `MetadataAppProvider.swift` | 2 秒超时，冷启动会短暂阻塞主线程 |
| HotKey 使用 Carbon API | `HotKeyManager.swift` | 已废弃 API，Command+Space 通常被系统占用 |
| 事件处理错误不上报 | `HotKeyManager.swift` | 错误仅打印，不对用户可见 |
| 无设置界面 | — | 快捷键、目录、外观均无法自定义 |
| 无文件搜索 | — | 仅支持应用搜索，缺乏文件与文档发现 |

## 三、后续开发计划

### Phase 1 — 基础完善（优先级：高）

> 目标：让现有功能更稳定、可配置，修复已知技术债

| # | 任务 | 描述 |
|---|------|------|
| 1-1 | 异步化 MetadataAppProvider | 用异步 API 替换阻塞 Semaphore，消除冷启动卡顿 |
| 1-2 | 快捷键配置 UI | 允许用户在设置界面自定义全局快捷键 |
| 1-3 | 应用发现目录配置 | 支持用户添加/删除自定义扫描目录 |
| 1-4 | 错误反馈 UI | 将快捷键注册失败等错误以更友好的方式呈现 |
| 1-5 | 设置面板骨架 | 建立 Settings UI 的基本框架，为后续配置项提供承载容器 |
| 1-6 | HotKey 现代化 | 评估并迁移至 `NSEvent.addGlobalMonitorForEvents` 替代 Carbon API |

### Phase 2 — 功能扩展（优先级：中）

> 目标：拓展搜索维度，让 Spotdark 成为真正的效率工具

| # | 任务 | 描述 |
|---|------|------|
| 2-1 | 文件搜索 | 通过 Spotlight / FSEvents 支持文件与文档搜索 |
| 2-2 | 最近项目区块 | 在结果列表顶部展示最近使用的应用与文件 |
| 2-3 | 分类分组显示 | 将结果按「应用」「文件」「命令」分组展示，提升可读性 |
| 2-4 | 网络搜索集成 | 输入无结果时提供"用浏览器搜索"选项（Google、百度等可配置） |
| 2-5 | 计算器 / 单位换算 | 识别数学表达式或单位格式，直接在面板内计算并显示结果 |
| 2-6 | 模糊匹配优化 | 引入 Levenshtein 或 N-gram 等算法改善拼写容错能力 |

### Phase 3 — 高级特性（优先级：低）

> 目标：差异化与平台深度集成，打造高级用户首选工具

| # | 任务 | 描述 |
|---|------|------|
| 3-1 | 插件 / 扩展系统 | 允许第三方开发者通过定义好的协议扩展命令和搜索源 |
| 3-2 | 主题自定义 | 提供浅色/深色/自定义调色方案，支持系统跟随 |
| 3-3 | 剪贴板历史 | 记录最近 N 条剪贴板内容，可在面板中搜索并粘贴 |
| 3-4 | 固定项 / 收藏夹 | 支持拖拽或快捷键将常用应用置顶 |
| 3-5 | 窗口管理命令 | 提供「隐藏当前窗口」「全屏切换」等系统级命令 |
| 3-6 | 应用快速切换 | 类似 Cmd+Tab，在面板内显示运行中的应用列表 |
| 3-7 | 配置文件导入导出 | 用户设置可以导出备份，跨机器同步 |

## 四、UI/UX 改善计划

| 优先级 | 改善点 | 描述 |
|--------|--------|------|
| 高 | 搜索无结果提示 | 当搜索结果为空时显示友好提示，而非空白列表 |
| 高 | 加载动画 | 首次索引时显示加载状态，避免用户感到应用无响应 |
| 中 | 结果图标加载优化 | 图标异步懒加载，避免列表滚动时卡顿 |
| 中 | 快捷键提示 | 在搜索栏右侧展示当前全局快捷键 |
| 中 | 面板位置记忆 | 用户拖动面板后记住位置，下次打开在同处显示 |
| 低 | 动画调优 | 优化面板出现/消失动画，与系统动效风格统一 |
| 低 | Accessibility | 增加 VoiceOver 支持，确保无障碍访问 |
